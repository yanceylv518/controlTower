# M2 阶段点：浏览器走查清单（人工验证，~1 小时）

**准备**（10 分钟）：M1 验证同款环境——起 MySQL + Server（`make build-server`，env 同 M1 阶段验证）→ `cd webapp && pnpm install && pnpm build` → 浏览器开 `http://127.0.0.1:8080/`。建议先跑一遍 `deploy/e2e-server.sh` 造出实例/告警/命令/审计数据。

逐项打勾；任何一项不符，记下页面与现象反馈给 Claude。

## 入口与认证
- [ ] `/` 直达登录页；错误密码提示可读；连错 5 次提示锁定
- [ ] 登录成功进总览；刷新任意深链（如 `/alerts`）不 404；`/next/alerts` 自动跳转 `/alerts`
- [ ] 顶栏显示用户名；退出后回登录页，回退按钮不能进入页面

## 总览
- [ ] KPI 卡数值与趋势图渲染；时间范围切换生效；当前告警区随 e2e 数据显示
- [ ] 无实例的全新库（可选验证）：出现"前往实例管理"引导条
- [ ] 静置 30 秒自动刷新（最后刷新时间变化）；切走标签页再回来立即刷新

## 只读页（B2 遗留项在此补验）
- [ ] 客户/渠道/模型监控：左列表选中切换详情；详情历史趋势随 1h/6h/24h 切换；渠道页显示快照名字/状态/权重/模型 chips
- [ ] **多实例切换**（B2 未验证项）：建第二个实例并用 e2e 心跳造数据后，顶栏切实例，各页数据随之变化互不串
- [ ] 样本分析：筛选与上一页/下一页；系统状态四表含网络 RX/TX 列；用量统计三排行随时长切换
- [ ] **错误态重试**（B2 未验证项）：停掉 Server，页面出现错误态与重试按钮；重启 Server 后点重试恢复

## 操作页（B3 写操作数据流在此补验）
- [ ] 告警中心：对 e2e 造出的告警执行确认（带备注）→ 时间线抽屉出现 acknowledged 事件、actor 为你的用户名、note 正确；静默 30 分钟后状态变化
- [ ] 通知设置：建钉钉渠道（填 secret）→ 列表显示"已加签"且无明文；投递记录中 failed 行点重发状态变化
- [ ] 实例管理：创建实例走 Token 一次性弹窗（复制/警告/必须确认）；轮换 Token 显示宽限时间；停用后该实例 token 心跳失败（可用 curl 验）
- [ ] 渠道命令：详情页下发命令——未勾确认时按钮禁用；勾选下发后命令列表状态从 pending 流转（e2e 模拟回传后 succeeded）；审计页出现记录且操作人正确

## 设置与杂项
- [ ] 修改密码：旧密码错被拒；成功后被登出，新密码可登录（**验完记得改回**）
- [ ] 访问不存在路由显示 404 页；各页浏览器标签标题正确；favicon 显示
- [ ] 全程浏览器控制台无红色报错

## 结果

全部通过 → 回复"M2 阶段验证通过"，M2 正式关闭，进入 v2.0 发布准备（Agent 双模式接入方案 + 部署）。

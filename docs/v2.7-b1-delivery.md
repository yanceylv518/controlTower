# v2.7-B1 Web 体验修正交付说明

## 交付范围

本批次完成 `docs/codex-task-v2.7-b1-ux-fixes.md` 约定的六类体验修正，未修改 Agent，也未改变既有 API 字段语义。

1. **纯名称展示**：`metrics`、`metric-history` 增量返回 `display_name`，`display_key` 保持兼容；客户、渠道、模型维度列表和详情优先显示纯名称，原始维度放在 tooltip 或详情小字中。告警列表与总览告警同样隐藏名称尾部 ID，悬浮仍可查看原始值。
2. **指标顺序**：维度汇总卡、质量比例条、总览 KPI 和六张趋势图按任务优先级重排；P95 成为主延迟指标，P50/P99 收入副信息。
3. **七处分页**：告警中心、样本分析、用量统计、操作审计、通知投递、渠道命令、延时慢样本统一使用 20/50/100 条分页，默认 20 条；切换筛选条件回到第一页。慢样本接口只增 `offset` 参数，并在关联过滤后分页。
4. **延时默认实例**：首次进入延时分诊且没有全局实例时，优先选择存在在线 Agent 的第一个实例，否则选择实例列表第一项；只有选中实例后确实无 timing 数据才展示采集配置空态。
5. **维度页布局**：左列固定 300px 并独立滚动，搜索、状态筛选和视图切换吸顶；默认渲染前 50 项，可按需显示全部；详情独立滚动，窄屏恢复上下布局。
6. **健康墙与大列表**：健康墙限制高度并滚动，名称使用纯展示名，原始 ID 保留在 tooltip；维度行压缩高度，避免 200+ 项导致整页卡顿和双纵向滚动条。

## 兼容性

- `display_name` 是新增字段，旧客户端可忽略；`display_key`、`dimension_key` 未改变。
- 慢样本 `offset` 是可选参数，默认 0；旧调用行为不变。
- 不涉及数据库迁移，不涉及 Agent 配置或二进制更新。

## 已知限制

- 列表接口当前不返回总数，分页按“当前页满额则允许下一页”的方式工作；任务明确不引入 Server `COUNT`。
- 维度列表采用前 50 项渐进展开，不引入虚拟滚动。

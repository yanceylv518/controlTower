# v2.3-B1 Web 体验打磨交付说明

日期：2026-07-15

## 本批改动

- Server 新增集中式 `nameResolver`，按实例、渠道和用户解析名称，使用 60 秒进程内缓存；缺失名称时回退原 ID 文案。
- Dashboard API 保留所有旧字段，增量增加：metrics/metric-history 的 `instance_name`，alerts 的 `instance_name`/`display_key`，channel-commands、operation-audits、channel-snapshots 的 `instance_name`，audits 同时补充 `instance_id`。
- 侧栏 13 个入口接入 Element Plus 官方图标，增加 active 左侧高亮条。
- 客户/渠道/模型共用维度页增加请求量/错误排序、错误率红橙绿状态点和四条比例指标的明确标签。
- 总览卡片增加图标和成功率/错误率/运行异常的颜色语义，请求数与 TPM 使用千分位，告警优先显示可读维度名称。
- 操作审计改为显示实例名，原实例 ID 通过悬停提示保留；全站格式化能力集中到 `utils/format.ts`。
- 新增 `b7.css`，统一侧栏图标间距、比例标签、状态点、卡片色板和窄屏布局。

## 页面走查清单

建议入口：`http://<control-tower-host>:8080/login`，登录后按以下顺序检查：

1. 总览：语义色 KPI、千分位、趋势空态、告警显示名。
2. 客户监控：用户名、排序切换、错误率状态点、四条指标标签。
3. 渠道监控：渠道名与 ID、渠道状态、模型标签和渠道操作。
4. 模型监控：模型名称、趋势和比例指标。
5. 告警中心：告警状态、维度显示名和时间线。
6. 样本分析：用户/模型/Token/错误摘要。
7. 用量统计：三类维度显示名及用量数据。
8. 延时分诊：实例选择、趋势、慢样本和 Bytes。
9. 系统状态：Agent、系统指标、健康检查与容器状态。
10. 通知设置：渠道状态、投递记录与重发操作。
11. 实例管理：实例名称、Agent 在线状态和 Token 轮换。
12. 操作审计：实例名称（悬停显示 ID）、目标和操作时间。
13. 设置：密码修改流程。

演示数据可按 `deploy/seed-demo-data.sh` 初始化；最终视觉验收以用户浏览器逐页走查为准。

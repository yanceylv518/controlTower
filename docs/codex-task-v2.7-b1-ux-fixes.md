# Codex 任务：v2.7-B1——Web 体验修正（用户名/指标排序/分页/布局/延时分诊默认态）

用户走查反馈六项,全部展示与交互层,零 Agent 改动、API 只增不改。

**文末自查清单粘贴进 commit message;禁止 force push;交付前 Linux 跑全量测试。**

## 工作项

### 任务 1：客户监控不直接显示用户 ID

- Server：metrics/metric-history 的响应新增 `display_name` 字段（纯名称,无 "(ID n)" 后缀;解析不到名称时回退 `用户 12` 全串）——`display_key` 保持现状不动（别的页面在用）;
- 前端：维度页左列、详情标题、健康墙、告警跳转落点等显示 `display_name`,原始 ID 放悬停 tooltip 与详情头部小字;渠道/模型维度同样处理（渠道显示纯名称,ID 进 tooltip）。

### 任务 2：指标按重要性重排

- 维度页详情指标卡顺序：**请求数、错误率、成功率、P95** 第一排;**TTFT、缓存命中率、Token In/Out、Quota** 第二排;P50/P99 并入延迟卡副行;
- 质量比例条顺序：错误率、成功率、缓存 Token 占比、流式占比;
- 总览 KPI 顺序：成功率、错误率、请求数、P95、TPM、平均耗时、健康、容器;
- 趋势图顺序：请求与错误 → 延迟 → 成功率/错误率 → TTFT → 缓存命中率 → Token。

### 任务 3：全站列表分页（含"数据多了就卡"）

统一方案（**零 Server 改动**）：前端 `el-pagination`（prev/next + 每页 20/50/100,默认 20）,请求带 `limit=pageSize, offset=page*pageSize`;`items.length < pageSize` 时禁用下一页;切过滤条件回第一页。覆盖页面：告警中心、样本分析、用量统计、操作审计、通知投递记录、渠道命令、延时分诊慢样本（该接口已有 limit,若缺 offset 补之,additive）。默认页大小 20 让首屏请求量直接降一个量级。

### 任务 4：延时分诊默认态

- 进入 `/latency` 时若未选实例,自动选中实例列表第一个（在线优先）;只有一个实例时不再要求手选;
- 空态文案仅在"选中实例确实无 timing 数据"时出现,并保留配置引导;
- 全局实例选择器行为不变（用户手动切换照常）。

### 任务 5：维度页布局重构

- 左列固定宽度(约 300px)、**独立滚动**,搜索框+状态签+排序吸顶（sticky）;行高压缩(一行名称+一行数据);
- 列表默认渲染前 50 项 + 底部"显示全部 (N)"按钮（配合搜索/过滤,大多数场景不用点开）;
- 右侧详情区独立滚动;**整页禁止出现双纵向滚动条**;≤900px 退化为上下布局;
- 健康墙同样限高滚动。

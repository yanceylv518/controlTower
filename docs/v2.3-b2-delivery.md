# v2.3-B2 Web 打磨收尾交付说明

日期：2026-07-15

## 已实现

- 告警补充维度类型、维度键、实例名和可读名称；告警中心与总览支持一键跳转到对应客户、渠道或模型，并通过 `?key=` 自动选中。
- P95 告警摘要显示窗口请求数；命中直方图最高档时显示“≥60s（超出直方图量程）”，避免伪精确值。
- 维度详情卡片按当前 1h/6h/24h 选择展示区间聚合总数，不再把最新一分钟误写成区间总量。
- 渠道工作区加入实时搜索、五类状态过滤、异常优先排序、无流量/已禁用折叠组，以及可记忆的健康墙视图；窄屏自动回退列表。
- 样本分析统一时间、数字和耗时格式；用量统计统一千分位；通知投递统一时间与长错误摘要展示。
- 系统状态页既有卡片、三类趋势图、陈旧标记与折叠原始表已核对保留。
- 全站定时刷新与页面重新可见时改为静默刷新：已有数据持续显示，后台失败保留旧数据；维度历史图同样原地更新，不再每 30 秒闪烁重建。

## API 兼容性

- `AlertItem` 仅新增 `dimension_type`、`dimension_key`，不修改原字段。
- `GET /api/dashboard/metric-history` 仅新增可选 `aggregate=true`；默认响应行为不变。
- Agent 无改动，无新增依赖。

## 人工验收建议

使用真实上报数据依次检查告警名称与跳转、1h/6h/24h 汇总数、样本格式，以及渠道搜索、状态筛选、折叠组和健康墙记忆。健康墙最终视觉以浏览器实测为准。

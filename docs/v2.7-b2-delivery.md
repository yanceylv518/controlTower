# v2.7-B2 系统设置中心交付说明

系统设置现支持数据库覆盖、环境变量和内置默认值三级回退，优先级为 `DB > env > default`。Provider 缓存 60 秒；页面保存后会立即失效缓存，下一轮任务直接读取新值，无需重启 Server。

| 配置 | 默认值 | 范围 |
| --- | ---: | --- |
| 明细 / metric_5m / 运行态保留天数 | 30 / 90 / 7 | 1–365 天 |
| Agent 离线时间 | 120 | 正整数秒 |
| CPU 警告 / 严重 | 80 / 90 | 1–100，警告小于严重 |
| 内存警告 / 严重 | 80 / 90 | 1–100，警告小于严重 |
| 磁盘警告 / 严重 | 85 / 95 | 1–100，警告小于严重 |
| 错误率警告 / 严重 | 20 / 50 | 1–100，警告小于严重 |
| P95 警告 / 严重 | 5 / 10 | 0.5–600 秒，警告小于严重 |
| 告警通知总开关 | true | true / false |

`GET/PUT /api/dashboard/settings` 均受登录保护；PUT 会记录 `settings.update` 操作审计。PUT 采用完整替换语义：页面提交当前覆盖值；若要恢复环境变量或默认值，可在数据库删除对应键（或通过 API 提交不包含该键的完整集合）。

迁移 `011_system_settings.sql` 仅新增表，不修改或重建既有表。五个动态接线点为保留清理、离线判定、CPU/内存/磁盘阈值、错误率/P95 阈值和通知总开关。

验证：`go test ./...`；`cd webapp && pnpm build`；011 迁移无 `ALTER TABLE` 反向断言。

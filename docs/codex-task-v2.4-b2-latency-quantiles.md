# Codex 任务：v2.4-B2——延迟分位数桶内插值（修"阶梯状延迟图"）

> **状态：已被 v2.5-B1（f62e0cc）吸收交付,勿重复执行。** 插值实现在 internal/latencyhist（验收补了单调性/边界测试）;读侧回退语义见 v2.5-B1。

用户反馈维度页延迟图不合理。根因：`internal/latencyhist.Quantile` 直接返回所在桶上界,P50/P95/P99 只会取 `0.25/0.5/1/2/3/5/10/30/60/120` 十个值——曲线呈方波阶梯、8s→11s 显示成 10→30、超 60s 一律顶格 120 拉飞纵轴。本批把分位数改为**桶内线性插值**（histogram_quantile 标准做法）,纯读侧计算,不动 Agent、不动 schema,历史数据即刻受益。

**文末自查清单粘贴进 commit message。与 v2.4-B1 无依赖,可先做、单独提交。**

## 工作项

### 任务 1：`internal/latencyhist.Quantile` 改插值

- 定位目标桶后,在桶内做线性插值：`lower + (target − cumulativeBefore) / bucketCount × (upper − lower)`;首桶 lower=0;
- **最高桶（60~120]）**：同样插值到 120 为止;若分位点落在最高桶,结果仍可能贴近 120——这是直方图量程极限,精确化归下批（Agent 直报精确分位数）;
- 单调性天然保证（同一直方图上 q 越大值越大）,加测试断言 P50≤P95≤P99;
- 边界测试：空桶、全部落一桶（桶内均匀插值）、target 恰在桶边界、单样本。

### 任务 2：影响面核对（同一函数的所有调用点行为都变,逐个确认合理）

- `metric_handler`（维度页 P50/P99）、`aggregator`（Agent 侧写 metric_1m 的 p95——**注意**:该值是 Agent 二进制算的,本批只改了库函数,**生产 p95 列要等 Agent 换二进制才变精细**,交付说明写明;Server 读侧的 P50/P99 即刻生效）;
- 调权引擎 evidence 里的 p95（取 MAX(p95_use_time),粒度变化无碍,确认无阈值依赖桶边界值）;
- P95 告警阈值 5s/10s（`alert_handler`）:插值后告警触发点会更精确（原来 4.2s 会被显示为 5 触发边缘抖动）,确认阈值语义不需调整,交付说明记录一句;
- 既有测试里所有断言桶上界精确值（如 `==120`、`==1.5`）的地方按插值语义更新。

### 任务 3：图表微调（可选,顺手）

维度页延迟图 tooltip 值保留两位小数;若 P99 与 P95 曲线因量程极限长期贴合 120,不做特殊处理（等精确化批次）,但 y 轴改 `max: 'dataMax'` 语义确认不被 120 顶格值绑架（现状如此则不动）。
